LAYER 3 OF THE OSI MODEL

The 3 of the osi model is the network layer. It is reponsible for routing and fowarding data packets between different networks. this layer uses logical addressing (such as IP addresses) to determine the best path for data packetsto reach their destiantion.

Key features Of The Network Layer Include:

  1. Routing: The network layer determines the best path for data packets to travel between different networks. This involves making decisions based on network topology,congestions levels, and other factors.
  2. Addressing: The network layer assigns logical addresses to devices on a network, such as IP addresses. These addresses are used to identify devices and determine where data packets should be sent.
  3. Fragmentation and reasssembly: The network layer is responsible for breaking down data packets in to smaller segments for transmission across a network, and then reassembling them at the destination.
  4. Error handling: The network layer may also include error detection and correction mechanisms to ensure that data packets are delivered accurately.
  5. Quality of service (QoS): The network layer can priorize certain types of traffic over others, such as giving priority to voice or video to ensure a better user experience.

Network Layer Protocols

  1. IP(Internet Protocol)
  2. ICMP(Internet Control Messaging Protocol)
  3. ARP(Address Resolution Protocol)

LAYER 4 OF THE OSI MODEL

The layer 4 protocol of the OSI model is the Transport layer.This layer is responsible for ensuring that the data is delivered in the correct order and retransmitted if errors occur.

Key Services Of The Transport Layer

  1. Connection-Oriented Communication: Establishes a connection between devices before data is sent,ensuring that data is delivered in the correct order and retransmitted if errors occur.
  2. Connectionless Communication: Sends data without establishing a connection first, but still provides error detection and correction.
  3. Flow Control: Regulates the amount of data send to prevent network congestion.
  4. Error Control And Correction: Detects and corrects errors that occur during data transmission
  5. Multiplexing And Demultiplexing: Allows multiple applications to share the same connection and enables multiple devices to communicate over the same network.
The Transport layer plays a crucial role in ensuring that data is delivered reliably and efficiently over a network,and is an essential part of the OSI model.

Transport layer protocols

  1. Transmission Control Protocol(TCP:A connection-Oriented protocol that ensures reliable data transfer.
  2. User Datagram Protocol(UDP):A connectionless protocol that provides best-effort delivery of data.
  3. Datagram congestion Control Protocol(DCCP):A connection-oriented protocol that provides congestion control and reliability for real-time applications.
  4. Internet Layer(IL):A protocol that provides connection-oriented and connectionless services for the transport layer
